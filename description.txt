OPENAPI спецификация лежит в папке openapi.
Сервис предоставляет возможность:
1 - создавать нового пользователя
2 - авторизовываться при наличии валидного username и password
3 - просматривать список всех зарегестрированных пользователей
4 - отправлять заявку в друзья любому пользователю
5 - отклонять заявку в друзья
6 - отзывать отосланную заявку
7 - принимать заявку - переходить в статус дружбы с пользователем
8 - смотреть список отправленных заявок
9 - смотреть список входящих заявок
10 - смотреть список друзей
11 - удалить пользователя из друзей
Для проверки работоспособности требуется поднять БД. Были созданы fixtures для этого.
В начале требуется создать миграции,если их нет:
./manage.py makemigrations
./manage.py migrate
После чего требуется прописать:
./manage.py loaddata contacts.fixtures.{requests.json} и {users.json}
Произойдет загрузка сохраненных данных. На них можно проверять работу приложения.
Для запуска можно использовать dockerfile, прописав:
docker-compose up -d
И перейдя на localhost:8000 попасть на стартовую страницу.
Можно осуществить запуск командой:
./manage.py runserver
Не было посвящено много времени внешнему виду, поскольку задание всё же состояло в написании backend-а.
Поэтому всё выглядит, прямо скажем примитивно.
Для авторизации можно использовать любой username из представленных:
 maxmill,frmill,petmill,troymill,tmill,admin,dmill
Пароль для всех пользователей одинаков: qwer123456
Описанные в задании статусы реализованы на странице /users где происходит отображение всех пользователей
Рядом с каждым пользователем написан текущий статус с возможностью его изменить
возможным способом в зависимости от текущего статуса
Их просмотр невозможен без авторизации
на странице /profile реализован профиль зарегестрировавшегося пользователся с возможностью его менять
Там же расположен список друзей и исходящих и входящих заявок в друзья с возможностью отклонить, принять и т д.
Также реализованы простые тесты в файле contacts/tests.py
